股癌追蹤資料字典
版本：0.2
日期：2026-06-16

一、文件目的

本文件定義 data/processed/ 目錄下核心 CSV 的欄位、型別、允許值與使用規則。

核心 CSV：

1. episodes.csv
   記錄節目、影片、文章或社群貼文等來源單位。

2. mentions.csv
   記錄從來源內容中抽出的公司、股票、產業或主題觀點。

3. instruments.csv
   記錄股票、指數、產業概念或其他可追蹤標的。

4. concept_proxies.csv
   記錄概念型 mention 對應的可交易代理籃子。

5. mention_returns.csv
   記錄每筆 approved mention 的報酬計算結果。

輔助 CSV：

1. transcript_inputs.csv
   記錄待轉錄音檔、來源 episode 與預計逐字稿輸出路徑。

2. audio_downloads.csv
   記錄已下載到本機的音檔路徑、大小與 hash。音檔本身不進 Git。

3. audio_chunks.csv
   記錄已切分的音檔段落。chunk 檔案本身不進 Git。

4. transcription_runs.csv
   記錄 ASR 轉錄執行結果。逐字稿原文不進 Git。

5. data/prices/*.csv
   記錄 scripts/fetch_prices.py 抓取的每日價格資料。

報表與人工工作表：

1. reports/pending_mentions_review.csv
   記錄待人工校對的 pending mentions、短證據、逐字稿行號提示與審核決策欄位。

2. reports/approved_company_bullish_returns.csv
   記錄已 approved 且可追蹤績效的公司與概念 proxy 追蹤標的。檔名沿用第一版，但內容已改為「每個 ticker / proxy 一列」：報酬從第一次提及開始計算，後續提及日期列在同一列。

3. reports/summary.csv
   記錄目前資料管線的總覽指標，例如 episode 數、mention review 狀態、return 狀態、可得 horizon 與平均報酬。

4. reports/concept_proxy_review.csv
   記錄概念 proxy baskets 的人工審核工作表，協助檢查代理成分、權重與待修正項目。

5. reports/dashboard.html
   靜態測試版 Dashboard，整合 summary、return report 與 concept proxy review 的可讀視圖。

6. docs/index.html
   GitHub Pages 發布入口，內容由 scripts/build_dashboard.py 從相同 Dashboard HTML 同步產生。

7. reports/new_episodes.csv
   記錄 RSS 同步時新加入的集數與下一步處理動作。

8. reports/auto_stock_mentions.csv
   記錄 scripts/auto_extract_stock_mentions.py 掃描逐字稿後的股票命中、推定 stance、證據片段與處理動作。

本機設定檔：

1. .env
   用於儲存 OPENAI_API_KEY 等本機密鑰。此檔案已被 .gitignore 排除。

2. .env.example
   範例格式，可 commit，但不得包含真實 key。

第一版採 CSV，是為了快速建立可手動操作的 MVP。未來若資料量增加，可轉成 SQLite 或 PostgreSQL。


二、通用規則

1. ID 規則
   - episode_id 使用 ep_ 前綴，範例資料使用 sample_ep_ 前綴。
   - mention_id 使用 m_ 前綴，範例資料使用 sample_m_ 前綴。
   - instrument_id 使用 inst_ 前綴。

2. 日期格式
   - 日期使用 YYYY-MM-DD。
   - 若未來需要時間，另增 published_time 或 published_at_time，不在日期欄混用。

3. 空值規則
   - 未知但未來可補：留空。
   - 無此欄位意義：留空。
   - 判斷不明：使用 unclear。
   - 市場未知：使用 unknown。

4. 布林值
   - 使用 true / false 小寫文字。

5. 範例資料
   - sample_ 開頭的資料只用於驗證欄位。
   - 正式統計或報表應排除 sample_ 開頭的資料。


三、episodes.csv

用途：
記錄每一個來源單位。一集 Podcast、一支 YouTube 影片、一篇文章或一則社群貼文都算一筆 episode。

欄位：

1. episode_id
   型別：string
   必填：是
   說明：來源單位唯一 ID。
   範例：ep_20260605_001

2. source_name
   型別：string
   必填：是
   說明：節目或來源名稱。
   範例：股癌 Gooaye

3. source_type
   型別：enum
   必填：是
   允許值：podcast / youtube / threads / article / manual
   說明：來源類型。

4. title
   型別：string
   必填：是
   說明：節目、影片、文章或貼文標題。

5. published_at
   型別：date
   必填：是
   說明：來源發布日期。

6. url
   型別：string
   必填：否
   說明：來源 URL。

7. duration_seconds
   型別：integer
   必填：否
   說明：音訊或影片長度秒數。文章或 Threads 可留空。

8. transcript_status
   型別：enum
   必填：是
   允許值：missing / manual / imported / generated / reviewed
   說明：逐字稿取得狀態。

9. notes
   型別：string
   必填：否
   說明：補充說明。


四、mentions.csv

用途：
記錄某一段內容中形成的股票、公司、產業或主題觀點。

欄位：

1. mention_id
   型別：string
   必填：是
   說明：mention 唯一 ID。
   範例：m_20260605_001

2. episode_id
   型別：string
   必填：是
   說明：對應 episodes.csv 的 episode_id。

3. published_at
   型別：date
   必填：是
   說明：來源發布日期，用於回測基準日。

4. company_or_theme
   型別：string
   必填：是
   說明：公司、股票、產業、指數或主題名稱。

5. ticker
   型別：string
   必填：否
   說明：股票代號。概念或產業可留空。

6. market
   型別：enum
   必填：是
   允許值：TWSE / TPEx / US / HK / concept / unknown
   說明：市場或標的類型。

7. mention_type
   型別：enum
   必填：是
   允許值：company / industry / theme / index / commodity
   說明：mention 標的類型。

8. stance
   型別：enum
   必填：是
   允許值：bullish / bearish / neutral / watch / past_review / unclear
   說明：方向性判斷，定義見 docs/標註規則.txt。

9. conviction
   型別：enum
   必填：是
   允許值：low / medium / high
   說明：觀點強度。

10. time_horizon
   型別：enum
   必填：是
   允許值：short / medium / long / unclear
   說明：觀點時間尺度。

11. evidence_text
   型別：string
   必填：是
   說明：支持標註的原文短句。approved mention 不可空白。

12. rationale
   型別：string
   必填：是
   說明：人工或 AI 對標註原因的簡述。

13. review_status
   型別：enum
   必填：是
   允許值：pending / approved / rejected / needs_context
   說明：人工審核狀態。

14. reviewer_note
   型別：string
   必填：否
   說明：審核補充說明。

正式追蹤納入條件：

1. review_status = approved。
2. evidence_text 不可空白。
3. 若 mention_type = company，ticker 不可空白，且會納入後續價格表現追蹤。
4. 若 market = concept，需有啟用的 concept proxy，且 stance = bullish 或 bearish，才納入 proxy 績效追蹤。
5. mention_id 不可以 sample_ 開頭。

方向性績效解讀：

1. bullish / bearish 代表明確方向性看法，可用於方向命中或超額報酬評估。
2. neutral / watch / past_review 仍可追蹤後續價格表現，但不得解讀成新的看多或看空訊號。
3. auto_ 開頭的 mention_id 代表由逐字稿別名與語氣關鍵字自動產生，不需人工審核。

六、concept_proxies.csv

用途：
記錄概念型 mention 的代理籃子。每一列是一個概念中的一個成分股；同一個 concept_name 可以有多列。第一版以人工 v0.1 等權籃子為主，目的是讓概念題材可以先進入追蹤，不代表投資建議。

欄位：

1. concept_name
   型別：string
   必填：是
   說明：對應 mentions.csv 的 company_or_theme。

2. proxy_id
   型別：string
   必填：是
   說明：代理成分唯一 ID。

3. ticker
   型別：string
   必填：啟用列必填
   說明：代理成分股票代號。

4. market
   型別：enum
   必填：啟用列必填
   允許值：TWSE / TPEx / US / concept
   說明：代理成分市場。is_active=true 時必須是 TWSE / TPEx / US。

5. name
   型別：string
   必填：啟用列必填
   說明：代理成分名稱。

6. weight
   型別：number
   必填：是
   說明：代理成分權重。同一概念的啟用權重第一版應加總為 1。

7. benchmark_market
   型別：enum
   必填：啟用列必填
   允許值：TWSE / TPEx / US
   說明：概念籃子用來比較的 benchmark 市場。

8. is_active
   型別：boolean
   必填：是
   說明：是否納入概念 proxy 計算。

9. notes
   型別：string
   必填：否
   說明：代理籃子的人工備註、版本或限制。

第一版計算規則：

1. 只有 review_status=approved、stance 為 bullish / bearish，且 concept_name 有啟用 proxy 的概念 mention 會進入 proxy 回測。
2. 概念 proxy 使用各成分股的 adj_close 計算報酬，再依 weight 做加權平均。
3. concept mention 的 base_price 在 mention_returns.csv 中記為 1，代表合成籃子基準淨值。
4. benchmark 依 benchmark_market 決定，目前 TWSE 使用 ^TWII，US 使用 ^IXIC。
5. 若概念沒有明確上市代理，應保留 is_active=false，不強行計算。


七、mention_returns.csv

用途：
記錄每筆 mention 在指定期間的報酬與相對 benchmark 表現。

欄位：

1. mention_id
   型別：string
   必填：是
   說明：對應 mentions.csv 的 mention_id。

2. ticker
   型別：string
   必填：否
   說明：股票代號。概念型 proxy mention 會記為 concept_proxy:{概念名稱}。

3. market
   型別：enum
   必填：是
   允許值：TWSE / TPEx / US / HK / concept / unknown

4. stance
   型別：enum
   必填：是
   允許值：bullish / bearish
   說明：主要回測只處理 bullish / bearish。

5. base_trade_date
   型別：date
   必填：否
   說明：發布後第一個交易日。

6. base_price
   型別：number
   必填：否
   說明：基準日收盤價。

7. current_trade_date
   型別：date
   必填：否
   說明：目前累計報酬使用的最新價格日期。

8. current_price
   型別：number
   必填：否
   說明：current_trade_date 的調整後收盤價。概念 proxy 會記為合成籃子淨值。

9. current_return
   型別：number
   必填：否
   說明：從 base_trade_date 到 current_trade_date 的標的實際報酬率，不做 bullish / bearish 方向反轉。

10. benchmark_current_return
   型別：number
   必填：否
   說明：benchmark 從 base_trade_date 到 current_trade_date 附近最新可得交易日的報酬率。

11. excess_current_return
   型別：number
   必填：否
   說明：current_return - benchmark_current_return。

12. return_7d
   型別：number
   必填：否
   說明：7 天後報酬率。使用小數，例如 0.05 代表 5%。此欄沿用既有回測邏輯，bearish 會轉成方向調整後報酬。

13. return_30d
   型別：number
   必填：否
   說明：30 天後報酬率。

14. return_90d
   型別：number
   必填：否
   說明：90 天後報酬率。

15. return_180d
   型別：number
   必填：否
   說明：180 天後報酬率。

16. benchmark_return_7d
   型別：number
   必填：否
   說明：benchmark 7 天後報酬率。

17. benchmark_return_30d
   型別：number
   必填：否
   說明：benchmark 30 天後報酬率。

18. benchmark_return_90d
   型別：number
   必填：否
   說明：benchmark 90 天後報酬率。

19. benchmark_return_180d
   型別：number
   必填：否
   說明：benchmark 180 天後報酬率。

20. excess_return_7d
   型別：number
   必填：否
   說明：return_7d - benchmark_return_7d。

21. excess_return_30d
   型別：number
   必填：否
   說明：return_30d - benchmark_return_30d。

22. excess_return_90d
   型別：number
   必填：否
   說明：return_90d - benchmark_return_90d。

23. excess_return_180d
   型別：number
   必填：否
   說明：return_180d - benchmark_return_180d。

24. calculation_status
   型別：enum
   必填：是
   允許值：pending_price / calculated / missing_price / not_applicable / error
   說明：報酬計算狀態。

25. notes
   型別：string
   必填：否
   說明：錯誤原因或補充說明。

第一版計算規則：

1. 只計算 review_status=approved 的 mention。
2. mention_type=company 且 ticker 不空白的 mention 會進入單一公司價格表現追蹤。
3. concept / industry / theme 若有啟用 concept_proxies.csv proxy，且 stance 為 bullish / bearish，會進入概念 proxy 表現追蹤。
4. 沒有 ticker 的公司、沒有 proxy 的概念等 mention 會在 mention_returns.csv 中標為 not_applicable。
5. base_trade_date 使用 published_at 後第一個可取得的交易日。
6. 7 / 30 / 90 / 180 天報酬使用 base_trade_date 加上日曆天後第一個可取得的交易日。
7. 報酬使用 adj_close 計算；若來源沒有 adj_close，則使用 close。
8. benchmark 第一版：TWSE 使用 ^TWII，US 使用 ^IXIC。
9. 若未來 horizon 價格尚未出現，calculation_status 標為 pending_price。

十一、audio_chunks.csv

用途：
記錄長音檔切段後的 chunk metadata。OpenAI Speech-to-Text API 單檔上傳限制為 25 MB，因此長節目需先切段。
切段音檔只作為 ASR 輸入暫存素材；確認 chunk transcript 與合併逐字稿已產生後，即可刪除 data/raw/audio_chunks/ 中對應 episode 的 chunk 檔。

欄位：

1. episode_id
   型別：string
   必填：是
   說明：對應 episodes.csv 的 episode_id。

2. title
   型別：string
   必填：是
   說明：節目標題。

3. chunk_index
   型別：integer
   必填：是
   說明：chunk 順序，從 0 開始。

4. chunk_path
   型別：string
   必填：是
   說明：本機 chunk 檔案路徑。

十四、approved_company_bullish_returns.csv

用途：
記錄由 scripts/build_return_report.py 產生的追蹤標的報表。此報表從 mentions.csv、episodes.csv、concept_proxies.csv 與 mention_returns.csv 派生，用來檢查 approved company 標的，以及已啟用 proxy 的概念型 bullish / bearish 標的的基準價、可得 horizon 報酬與相對 benchmark 超額報酬。

此報表以追蹤標的為單位彙整：同一 ticker / market 的多次提及只保留一列，報酬欄位引用第一次提及的 mention_returns.csv 結果，後續提及日期與集數用清單欄位保留。若同一標的後續又被提到，應更新 mention_count、followup_published_dates、all_published_dates 與 mention_timeline，而不是新增第二列。

欄位群組：

1. tracking 與來源欄位
   欄位：tracking_id、first_mention_id、mention_ids、mention_count、first_episode_id、episode_ids、first_episode_title、episode_titles、first_published_at、followup_published_dates、all_published_dates、latest_published_at
   說明：用於追溯同一標的的第一次提及與後續提及。tracking_id 是報表層的標的 ID，不取代 mentions.csv 的 mention_id。

2. 標的與看法欄位
   欄位：company_or_theme、ticker、market、stance、stances、stance_timeline、conviction、time_horizon
   說明：company_or_theme、stance、conviction、time_horizon 取第一次提及的標註；stances 與 stance_timeline 保留後續提及的看法變化。

3. 基準價欄位
   欄位：base_trade_date、base_price
   說明：第一次提及發布後第一個可得交易日與該日價格。

4. 報酬狀態欄位
   欄位：available_horizons、calculation_status、return_notes
   說明：列出目前已可計算的 horizon，例如 7d；若後續價格尚未到期，calculation_status 會維持 pending_price。

5. 證據與時間線欄位
   欄位：evidence_text、evidence_items、mention_timeline
   說明：evidence_text 保留第一次提及的短證據；evidence_items 以 JSON list 保留每次提及的日期、集數、stance 與證據文字；mention_timeline 保留各次提及的日期、集數與 stance，方便檢查該標的是否真的值得納入追蹤。

6. 至今報酬欄位
   欄位：current_trade_date、current_price、current_return、current_return_pct、benchmark_current_return、benchmark_current_return_pct、excess_current_return、excess_current_return_pct
   說明：從第一次提及的 base_trade_date 到目前價格資料最後一日的標的實際漲跌與相對 benchmark 超額報酬。current_return 不做 bearish 方向反轉。

7. horizon 報酬欄位
   欄位：return_7d、return_30d、return_90d、return_180d，以及對應的 _pct 欄位
   說明：標的自身報酬；小數欄用於後續程式計算，_pct 欄用於人工閱讀。

8. benchmark 與超額報酬欄位
   欄位：benchmark_return_*、excess_return_*，以及對應的 _pct 欄位
   說明：benchmark 報酬與標的相對 benchmark 的超額報酬。

使用規則：

1. 此報表是派生檔，可由 scripts/build_return_report.py 重建。
2. 不在此報表中手動修改 mention 判斷；若要修正，應回到 mentions.csv 或 review workflow。
3. 每次重跑 scripts/fetch_prices.py 更新 mention_returns.csv 後，應再重跑 scripts/build_return_report.py。

十九、下一步檢查清單

建立或修改資料後，至少檢查：

1. CSV 欄位數是否與資料字典一致。
2. sample_ 開頭資料是否被排除在正式統計之外。
3. approved mention 是否都有 evidence_text。
4. bullish / bearish 且非 concept 的 mention 是否有 ticker。
5. mention_returns.csv 是否包含所有正式 approved mentions 或明確標記 sample。
6. instruments.csv 是否能支援常見別名查找。
7. transcript_inputs.csv 是否每筆都有 audio_url 與 transcript_path。
8. audio_chunks.csv 的 chunk_bytes 是否都低於 ASR 單檔上傳限制。
9. pending_mentions_review.csv 若有 review_decision，是否皆為允許值。
10. mention_returns.csv 中 calculated / pending_price rows 是否都有 base_trade_date 與 base_price。
11. data/prices/*.csv 是否每列欄位數一致。
12. approved_company_bullish_returns.csv 是否包含可追蹤的 approved company mentions，以及 approved、bullish / bearish 且有 proxy 的概念正式 mentions。
13. 每次更新價格後，是否已同步重建 approved_company_bullish_returns.csv。
14. summary.csv 是否包含 episodes.formal_total、mentions.formal_total、returns.formal_total 與 reports.approved_company_bullish_returns_rows 等核心指標。
15. concept_proxy_review.csv 是否與 concept_proxies.csv 的 proxy_id 完全一致。
16. dashboard.html 是否已由最新 reports/*.csv 重建。
17. new_episodes.csv 若有列資料，是否都已存在於 episodes.csv，且 next_action 為 download_audio。
