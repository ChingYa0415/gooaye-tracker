股癌追蹤操作流程
版本：0.2
日期：2026-06-16

一、文件目的

本文件記錄資料與腳本層的固定操作方式。若只是要更新價格、重建報表或驗證資料，優先看本文件，不必從完整規劃文件重新推導流程。


二、日常更新流程

用途：
更新價格、重算 mention_returns.csv、重建報酬檢查報表、重建總覽報表，並執行資料驗證。

完整更新：

python3 scripts/run_daily_update.py

只重建報表與驗證現有資料：

python3 scripts/run_daily_update.py --skip-price-fetch

指定價格日期區間：

python3 scripts/run_daily_update.py --start-date 2026-01-01 --end-date 2026-12-31

此流程會依序執行：

1. scripts/fetch_prices.py
2. scripts/build_return_report.py
3. scripts/build_summary_report.py
4. scripts/build_concept_proxy_review.py
5. scripts/build_dashboard.py
6. scripts/validate_data.py

輸出檔案：

1. data/prices/*.csv
2. data/processed/mention_returns.csv
3. reports/approved_company_bullish_returns.csv
4. reports/summary.csv
5. reports/concept_proxy_review.csv
6. reports/dashboard.html


二之一、新集數同步流程

用途：
從 Gooaye SoundOn RSS 檢查是否有比目前 episodes.csv 更新的集數，安全新增到 episodes.csv 與 transcript_inputs.csv，並輸出新集數工作表。

檢查但不寫入：

python3 scripts/sync_episodes.py --dry-run

正式同步：

python3 scripts/sync_episodes.py

輸出檔案：

1. data/processed/episodes.csv
2. data/processed/transcript_inputs.csv
3. reports/new_episodes.csv

預設行為：

1. 只新增比目前最新正式 episode 還新的 RSS items。
2. 不覆寫既有 generated / reviewed 等 transcript_status。
3. 不補舊集數；若要手動補歷史集數，才使用 --include-backfill。
4. 新集數的 next_action 會標為 download_audio。

週期性更新：

Codex automation「同步並分析股癌新集數」已設定為每週三與週六上午執行。任務會先跑 scripts/sync_episodes.py；接著檢查 data/processed/transcript_inputs.csv 中 transcript_status=missing 且尚未產生合併逐字稿的正式集數，依序下載音檔、切段、用本機 whisper.cpp 轉錄整集、合併逐字稿，接著執行 scripts/auto_extract_stock_mentions.py 與 scripts/run_daily_update.py。

此排程不需要人工審核才會把新標的納入追蹤。只要逐字稿中命中 instruments.csv 的單一股票別名，且上下文可判斷出 stance keyword，腳本會自動寫入 mentions.csv，review_status=approved，並在 reviewer_note 標記 no manual review。低信心或只有名稱、沒有語氣線索的命中會保留在 reports/auto_stock_mentions.csv，但預設不寫入正式 mentions。


三、資料驗證流程

用途：
檢查核心 CSV 欄位、ID 關聯、approved mention 規則、return rows、報表一致性、summary 核心指標與價格檔格式。

命令：

python3 scripts/validate_data.py

目前驗證重點：

1. data/processed/*.csv 欄位是否符合資料字典。
2. reports/*.csv 欄位是否符合派生報表規則。
3. episode_id、mention_id、instrument_id 是否重複。
4. mentions.csv 是否引用存在的 episode_id。
5. approved mention 是否有 evidence_text。
6. 正式 approved mentions 是否都有 mention_returns.csv row。
7. approved company mentions 是否都有報表 row；概念型 mentions 則只追蹤 bullish / bearish 且已啟用 proxy 的項目。
8. calculated / pending_price rows 是否有 base_trade_date 與 base_price。
9. data/prices/*.csv 是否有必要欄位與可解析的價格。
10. reports/summary.csv 是否包含必要總覽指標。
11. data/processed/concept_proxies.csv 的啟用代理成分是否有可抓價格的 ticker、market、權重與 benchmark_market。

若驗證失敗，先修正資料或重建派生檔，再重新執行。


四、新增或更新節目逐字稿流程

1. 更新 RSS transcript input manifest：

python3 scripts/fetch_transcript_inputs.py

2. 下載指定 episode 音檔：

python3 scripts/download_audio.py --episode-id ep_YYYYMMDD_XXX

3. 切分音檔：

python3 scripts/split_audio.py --episode-id ep_YYYYMMDD_XXX

4. 使用 OpenAI Speech-to-Text 轉錄：

python3 scripts/transcribe_openai.py --episode-id ep_YYYYMMDD_XXX

若 OpenAI API 額度不足，改用本機 whisper.cpp。未指定 --chunk-index 時，會轉錄該集所有 chunks；若只要補單段，可加上 --chunk-index：

python3 scripts/transcribe_whisper_cpp.py --episode-id ep_YYYYMMDD_XXX

5. 合併逐字稿：

python3 scripts/combine_transcripts.py --episode-id ep_YYYYMMDD_XXX

6. 確認 episodes.csv 與 transcript_inputs.csv 的 transcript_status 已更新。

注意：
逐字稿原文、原始音檔、切段音檔與模型檔都是本機 raw artifacts，不進 Git。確認合併逐字稿與 metadata 完成後，原始音檔與切段音檔可刪除。


五、自動抽取股票 mention 流程

用途：
不經人工審核，從已合併的逐字稿中自動整理有提到的單一股票、判斷股癌語氣，並納入後續績效追蹤。

正式執行：

python3 scripts/auto_extract_stock_mentions.py

檢查但不寫入：

python3 scripts/auto_extract_stock_mentions.py --dry-run

輸出檔案：

1. data/processed/mentions.csv
2. reports/auto_stock_mentions.csv

預設行為：

1. 只掃描 transcript_status=generated 的正式集數。
2. 只處理 instruments.csv 中可對應 ticker 且 market 為 TWSE、TPEx、US 的單一股票。
3. 同一集同一 ticker 已存在 mentions.csv 時不重複新增。
4. 有語氣關鍵字的命中自動寫入 mentions.csv，review_status=approved。
5. bullish / bearish / watch / past_review / neutral 會一併進入公司績效追蹤；其中 bullish / bearish 才代表方向性看法。
6. 沒有語氣關鍵字的低信心命中預設只寫入 reports/auto_stock_mentions.csv，action=skipped_low_confidence。
7. 若要把低信心 neutral 命中也寫入 mentions.csv，可加 --include-low-confidence，但目前不建議排程使用。

自動抽取後，執行：

python3 scripts/run_daily_update.py

此流程會更新價格、mention_returns.csv、追蹤報表、summary 與 dashboard。


五之一、候選 mention 與人工審核工具

下列工具保留給需要人工校對、補歷史資料或修正錯誤時使用；目前日常流程不依賴人工審核。

1. 掃描逐字稿中的標的別名：

python3 scripts/search_transcript_mentions.py --episode-id ep_YYYYMMDD_XXX

2. 人工將候選標的整理進 data/processed/mentions.csv，初始 review_status 使用 pending。

3. 產生人工審核工作表：

python3 scripts/build_review_queue.py

4. 在 reports/pending_mentions_review.csv 填寫 review_decision 與必要的 corrected_* 欄位。

5. 套用前先 dry-run：

python3 scripts/apply_review_decisions.py --dry-run

6. 正式套用：

python3 scripts/apply_review_decisions.py

7. 套用後重新驗證：

python3 scripts/validate_data.py

needs_context 複核：

1. 先查看 reports/pending_mentions_review.csv 的 transcript_line、transcript_context 與 reviewer_note。
2. 若是 ASR 專有名詞錯誤，使用較大的 whisper.cpp 模型重轉相關 chunk，輸出到 /tmp 或其他不進 Git 的暫存路徑。
3. 能確認標的與語氣時，在 review CSV 填入 approved、corrected_evidence_text 與 review_comment。
4. 無法確認標的或上下文時，填入 rejected，不強行納入正式資料。
5. 回寫後執行 scripts/run_daily_update.py，更新 mention_returns.csv、報表與 summary。


六、報酬與報表流程

單獨更新價格與 mention_returns.csv：

python3 scripts/fetch_prices.py

單獨重建報酬檢查報表：

python3 scripts/build_return_report.py

單獨重建總覽報表：

python3 scripts/build_summary_report.py

單獨驗證：

python3 scripts/validate_data.py

建議日常使用 scripts/run_daily_update.py，避免忘記重建報表或驗證資料。

概念 proxy 回測：

1. 在 data/processed/concept_proxies.csv 為概念型 mention 定義代理籃子。
2. concept_name 必須對應 mentions.csv 的 company_or_theme。
3. 每個啟用成分需填 ticker、market、name、weight、benchmark_market、is_active=true。
4. 第一版使用 manual v0.1 proxy；若沒有明確上市代理，保留 is_active=false。
5. 更新 proxy 後執行：

python3 scripts/run_daily_update.py

6. 概念型回測結果會寫入 mention_returns.csv，ticker 會標示為 concept_proxy:{概念名稱}。
7. reports/approved_company_bullish_returns.csv 會同時列出公司候選與已啟用 proxy 的概念候選，並以 ticker / proxy 為單位彙整；同一標的多次提及只出現一列，報酬從第一次提及開始計算，後續提及日期列在同一列。

概念 proxy 人工審核工作表：

python3 scripts/build_concept_proxy_review.py

輸出：

reports/concept_proxy_review.csv

此工作表會列出每個概念 proxy 成分、權重、benchmark_market、目前 directional mention 數、active proxy 數、權重加總與 concept_issue。人工審核時優先看 concept_issue 不是 ok 的列，例如 single_name_proxy、no_active_proxy、mixed_component_markets 或 mentioned_but_not_trackable。

目前 review_decision、corrected_weight、corrected_benchmark_market、corrected_is_active、replacement_* 與 review_comment 是人工填寫欄位。正式套用前仍需回到 data/processed/concept_proxies.csv 修改，或後續再建立 apply_concept_proxy_review.py。

Dashboard 測試版：

python3 scripts/build_dashboard.py

輸出：

reports/dashboard.html
docs/index.html

Dashboard 讀取 summary.csv、approved_company_bullish_returns.csv 與 concept_proxy_review.csv，提供總覽、追蹤明細、股癌看法、首次提及日期、提及次數、提及日期彈窗、證據來源彈窗、至今報酬、至今超額、7d 報酬條圖、7 / 30 / 90 / 180 horizon 狀態、公司 / 概念 proxy 篩選、狀態篩選、快速篩選、排序與 proxy 審核表。追蹤明細與 proxy 審核表欄頭採中文/英文格式。追蹤明細的標的、類型、看法、首次提及、基準日欄頭可直接點擊，依序切換原排序、升冪、降冪；其他排序仍可使用排序下拉選單。reports/dashboard.html 是本機檢查用靜態 HTML；docs/index.html 是 GitHub Pages 發布入口。

GitHub Pages 發布：

1. GitHub repo Settings -> Pages。
2. Source 選 Deploy from a branch。
3. Branch 選 main，Folder 選 /docs。
4. 儲存後，公開網址為 https://chingya0415.github.io/gooaye-tracker/。

注意：
至今報酬是從第一次提及後的 base_trade_date 到目前價格資料最後一日的標的實際漲跌，不做 bearish 方向反轉；7 / 30 / 90 / 180 horizon 欄位則沿用既有回測邏輯，也以第一次提及作為基準。


七、Git 提交前檢查

提交前至少執行：

python3 -m py_compile scripts/*.py
python3 scripts/run_daily_update.py --skip-price-fetch
git diff --check
git status --short

確認大型 raw artifacts、.env、模型檔、音檔與逐字稿沒有進入 Git。
