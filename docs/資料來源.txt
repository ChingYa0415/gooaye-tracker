一、Podcast 來源

1. Gooaye 股癌 SoundOn RSS

URL：
https://feeds.soundon.fm/podcasts/954689a5-3096-43a4-a80b-7810b219cef3.xml

用途：
取得股癌節目的 episode metadata，包含標題、發布日期、SoundOn 播放頁 URL、guid 與節目長度。

目前使用：
已用於建立 data/processed/episodes.csv 第一批 10 集樣本。

注意：
RSS 只提供節目 metadata 與簡介，不等於完整逐字稿。正式 mention 標註仍需要逐字稿、音訊片段或人工摘錄的原文證據。

五、ASR 與音檔處理工具

1. ffmpeg

用途：
切分 Podcast 長音檔，避免單檔超過 ASR API 上傳限制。

目前使用：
已安裝並用於將 EP667 音檔切成 4 個 MP3 chunk。

2. OpenAI Speech-to-Text API

官方文件：
https://platform.openai.com/docs/guides/speech-to-text

用途：
將音檔 chunk 轉成逐字稿。

目前設計：
scripts/transcribe_openai.py 預設使用 gpt-4o-mini-transcribe，呼叫 /v1/audio/transcriptions。

限制：
單檔上傳限制為 25 MB，因此長音檔需先透過 scripts/split_audio.py 切段。

目前狀態：
轉錄腳本已建立，但本機尚未設定 OPENAI_API_KEY，因此尚未執行實際 ASR。

本機設定方式：

1. 建立 .env 檔案。
2. 寫入 OPENAI_API_KEY=你的 key。
3. .env 已由 .gitignore 排除，不會推到 GitHub。

範例：
可參考專案根目錄的 .env.example。

注意：
不要把真實 API key 寫入 README、docs、CSV 或任何會 commit 的檔案。
.env 裡的值必須是真正的 OpenAI API key，通常以 sk- 開頭；不要保留 .env.example 裡的範例文字。

3. 合併逐字稿

工具：
scripts/combine_transcripts.py

用途：
把同一集的多個 chunk transcript 合併為 data/raw/transcripts/{episode_id}.txt。

注意：
逐字稿原文仍屬本機 raw artifact，不進 Git。

4. whisper.cpp

工具：
whisper-cli

安裝方式：
brew install whisper-cpp

模型：
data/models/ggml-base.bin

模型來源：
https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-base.bin

用途：
在 OpenAI API 配額不足或不想使用雲端轉錄時，使用本機 Whisper 產生逐字稿。

目前使用：
已用 scripts/transcribe_whisper_cpp.py 將 EP667 的 4 個 audio chunks 轉成逐字稿，並用 scripts/combine_transcripts.py 合併成 data/raw/transcripts/ep_20260603_667.txt。

注意：
模型檔、chunk transcript 與合併逐字稿皆為本機 raw artifact，由 .gitignore 排除，不進 Git。
